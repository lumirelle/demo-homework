3.2 环境就绪命令

# 1. 自检环境（首次或换机器） bun run check-env # 2. 启动数据库 + 缓存（首次拉镜像约 15s） bun run dev:up # 3. 验证 7 张表已初始化 docker-compose -f infra/docker-compose.dev.yml exec -T postgres \ psql -U ats_user -d ats_db -c "\dt" # 4. 启动后端（首次拉 Maven 依赖约 30s） bun run be:dev # 5. 启动前端 bun run fe:dev # 6. 验收 curl http://localhost:5173/api/v1/health # → {"code":0,"msg":"ok","data":{"db":"UP","redis":"UP"}}

3.3 容器命令兼容性

项目作者用的是 Podman 5.x + docker-compose v5 binary。 npm scripts 里统一用 docker-compose（v1 写法），Docker / Podman 都能跑。 若你只装了 docker compose v2（带空格），把 package.json 里 dev:* 脚本的 docker-compose 改成 docker compose 即可。

3.4 容易踩的坑

坑原因 / 解
schema 改了，但容器里还是旧表 pg 的 /docker-entrypoint-initdb.d 只在首次启动空库时执行。改 schema 后必须 bun run dev:reset（删 volume 重来）
后端连不上 Postgres 检查 ats-backend/.env 是否已从 .env.example 复制；DB_HOST 应该是 localhost，端口 5432
Lombok 在 JDK 25 上 deprecation warning 已升到 1.18.42 缓解；如有更新到 1.18.46+ 可彻底消除
JWT 密钥未生成 M1 启动前跑 pwsh -File scripts/new-jwt-keys.ps1，按提示填入 ats-backend/.env

5. M1 · 认证模块 已完成

验收记录（2026-05-22）

  • 后端实现：JwtService（RS256 keypair 加载 + access/refresh token 签发/校验）+ Spring Security STATELESS + JwtAuthenticationFilter + JwtAuthEntryPoint + AuthController（register/login/refresh/logout/me）+ AdminController（POST /admin/users 仅 ADMIN，role 受限 HR/CANDIDATE）+ PgEnumTypeHandler（user_role enum 映射）
  • 前端实现：Pinia auth store（accessToken 内存 + user）+ axios 拦截器（withCredentials + 401 自动 refresh + 并发队列防 race）+ 路由 guard（requiresAuth + roles + 已登录重定向）+ /login /register /403 页面 + AppNavbar 角色徽章下拉
  • 单测覆盖：43 case / < 10s 全绿（JwtServiceTest 8 + AuthServiceTest 17 + WebSecurityIntegrationTest 18）。覆盖 sign/verify happy/过期/篡改/错 issuer、register 重复邮箱、login 4 error path、refresh rotation + 失效场景清 cookie、logout 幂等、401 三态、@PreAuthorize 角色守卫（ADMIN 200 / HR 403 / CANDIDATE 403）、CSRF disable 回归。不依赖 PG/Redis/Docker
  • e2e 联调：13 条 curl 全过（token rotation 验证、logout 后 refresh 被拒、CANDIDATE 调 admin 403、重复注册 10004、role=ADMIN 越权 400）

关键决策与产物

  • Token 双轨：Access Token RS256 JWT / 15min（精简到 15min，比初设 2h 更安全），Refresh Token 随机 32B / 30d 存 HttpOnly Cookie；refresh SHA-256 后入 PG（非 Redis，避免增加依赖；Redis 留给后续 M2 缓存）
  • Rotation 策略：每次 refresh 立即 revoke 旧 token（防 token 复用攻击）；登录时 revoke 该用户所有旧 refresh（single-session）
  • 注册流程：CANDIDATE 自助注册 / HR 由 Admin 创建 / ADMIN 仅 DB seed（防权限提升）
  • 测试策略：见 SKILL 2.6.7 「后端测试分层」—— 三层分工（纯 JUnit / Mockito / @WebMvcTest），不引入 Testcontainers

踩坑

  • PostgreSQL user_role enum 默认 setString 报类型不匹配 → PgEnumTypeHandler 用 Types.OTHER（详见 ADR 2026-05-22）
  • JDK 25 + Spring Boot 3.4.0 默认 Mockito 5.14 不兼容（Could not modify all classes）→ 升级 Mockito 5.17 + byte-buddy 1.17 + surefire argLine（详见 ADR 2026-05-22）
  • Spring Boot 3.4 Mybatis-Plus BaseMapper 的 insert/updateById 同时声明 (T) 和 (Collection<T>) → Mockito argThat lambda 编译歧义 → 包一层 typed helper
  • JJWT 0.12 Claims 接口已无 setter → 用 new DefaultClaims(map) + put(Claims.SUBJECT, ...)

2026-05-22 UnoCSS 写法新增第三档：CSS 变量圆括号简写 bg-(--var) UnoCSS 原生支持 prop-(--var-name) ≡ prop-[var(--var-name)] 简写（圆括号在 HTML attribute name 中合法，不像方括号需要 attributify value）。比 bg-[var(--brand-500)] 短 7 字符，比 bg="[var(--brand-500)]" 短 9 字符；同样支持 hover variant（hover:bg-(--brand-700)）。新增规约：选择树第 3 档插入"单值 = CSS 变量 → 圆括号简写"，方括号仅保留给 arbitrary 字面值（max-w-[1200px] / tracking-[-0.05em]）。反模式 #7 新增：CSS 变量还在用方括号 + var() 写法是过时写法。渐变陷阱同步：bg-(--grad-xxx) 也会被当 background-color 静默失效，规避方式与方括号一致（用 rules 接管显式输出 background-image）。SKILL v0.0.15。
2026-05-22 样式方案改用 UnoCSS（Attributify 第一公民） 原子化 CSS + dead code 自动 purge；attributify mode 让模板里 bg="elevated" text="lg primary" 自带语义分组，可读性远超长 class 串。核心决策：CSS custom properties（tokens.css）是单一真值来源，UnoCSS theme 通过 var(--xxx) 引用，禁止重复定义色值，Naive UI / 原生 CSS / UnoCSS 三方共享一套真值。无值原子必须用 ~ 占位（如 flex="~" relative="~"），否则 vue-tsc 会把它当成 boolean attribute 报错。SKILL 2.5.5 沉淀 attributify 落地规约 + 反模式 + 何时仍保留 scoped style 的指引
2026-05-22 后端单测策略：三层分工（纯 JUnit / Mockito / @WebMvcTest），坚决不引入 Testcontainers MVP 阶段优先 跑得快、稳定、CI 无依赖。三层：(1) 纯 JUnit + @TempDir 测算法/工具类（如 JwtService 的 sign/verify/hash），用 helper 现场生成 RSA keypair 避免依赖 infra/jwt/；(2) Mockito 单测 覆盖 Service 业务分支（AuthService 17 case 含所有 error path）；(3) @WebMvcTest + @MockitoBean 启动真实 SecurityFilterChain 但 mock 掉 Service / Mapper，专测 401 / 403 / @PreAuthorize / CSRF 行为。不引入 H2 / Testcontainers：H2 不支持 PG enum + RETURNING + gen_random_uuid()，Testcontainers 拖慢 CI + 增加 Docker 依赖。SQL 正确性靠 dev compose 联调 + 后续 e2e 兜底。43 case < 10s。
2026-05-22 升级 Mockito 至 5.17 + byte-buddy 1.17（覆盖 Spring Boot 3.4.0 默认 5.14） JDK 24+（含 GraalVM 25.0.3 JVMCI）下 Mockito 5.14 的 byte-buddy 1.15 报 Could not modify all classes [class XxxService, class Object]，所有 inline mock 失败。诊断：mockito-inline 依赖 byte-buddy-agent 动态 attach 到当前 JVM 做 bytecode 重写；JDK 25 加强了对 self-attach 的限制（need -XX:+EnableDynamicAgentLoading），byte-buddy 1.17+ 才适配新的 ClassFile API。修复：pom.xml 显式声明 mockito.version=5.17.0 + byte-buddy.version=1.17.6 覆盖 Spring BOM；surefire argLine 加 -XX:+EnableDynamicAgentLoading -javaagent:.../byte-buddy-agent.jar -Xshare:off（最后一个关闭 CDS 防 instrumentation 冲突）。留给未来 agent 的信号：升级 Spring Boot 时若 BOM 的 mockito 已 ≥ 5.17 可移除 properties 覆盖，但 argLine 仍建议保留以兼容更新的 JDK。
